Motivation

GNNs Hit a Memory Wall on GPUs

🧱 The Bottleneck

GPUs are optimized for dense (GEMM) workloads. GNNs produce scatter–gather memory patterns that defeat HBM coalescing.

📦 The Scale Problem

ogbn-products: 2.4 M nodes, 100-dim features → ~960 MB feature matrix vs. 40 MB L2 cache on A100.

🔀 Root Cause

Random node ordering → connected nodes have distant indices → L2 capacity misses on every neighbor fetch.

Core Idea

Node Reordering = Lossless Locality Boost

🔑 The Key Insight

A permutation π of node indices → reordered adjacency A' = PAPT. GNN message-passing is equivariant to node relabeling — accuracy is guaranteed identical.

📐 Bandwidth Minimization

We seek π* that minimizes matrix bandwidth:
B(A) = max{|π(i) − π(j)|} for (i,j)∈E
Smaller bandwidth → neighbors at nearby indices → cache line reuse.

⚙️ Pre-processing, Not Training Change

Reordering is a one-time O(|E| log|E|) operation. No changes to model, loss, or optimizer.

Baseline Methods

RCM & METIS: Two Classic Approaches

Reverse Cuthill-McKee (RCM)

How it works

BFS-based traversal from a peripheral "seed" node, visiting neighbors in ascending degree order, then reversing the sequence. Produces a band-diagonal A'.

Result

Every non-zero lies within B positions of the diagonal. SpMM accesses rows of H in a narrow window of width B × d.

Complexity: O(|E| log|E|)
METIS Graph Partitioning

How it works

Multilevel graph partitioning into k balanced subsets V₁…Vₖ minimizing edge cut. Nodes within the same partition are grouped together.

Result

Block-diagonal A': most non-zeros fall within dense diagonal blocks of size ≈ n/k, fitting intra-partition features in L2.

EdgeCut = |{(i,j)∈E : separate partitions}|

⚠️ Prior work applies RCM and METIS independently and treats partition count k as a hyperparameter swept blindly. We do better.

πhybrid = πRCMlocal ∘ πMETIS

🎯 Why RCM alone isn't enough

Global RCM cannot exploit community structure. Large bandwidth remains due to cross-community edges.

🎯 Why METIS alone isn't enough

Internal block ordering is arbitrary. Non-zeros within each block are still scattered even if communities are grouped.

✨ Hybrid advantage

Local bandwidth Bp of subgraph G[Vₚ] ≪ global RCM bandwidth. Smaller graph → much tighter band.

✔️ Correctness preserved

Composition of permutations is a permutation. O(|E| log|E|) total complexity. Accuracy unchanged.

Novel Contribution 2

Cache-Size-Aware Partition Selection

💡 The Insight

For intra-partition SpMM to be fully cache-resident, the feature sub-matrix Hp ∈ ℝ|Vₚ|×d must fit in the L2 cache of capacity C bytes:

|Vₚ| · d · b ≤ C

📐 Derived Formula

Under balanced partitioning (|Vₚ| ≈ n/k), solving for k:

k* = ⌈ n · d · b / C ⌉

🔧 Practical Refinement

Add headroom factor α ∈ [1.0, 1.5] for inter-partition spill accesses:
k*_practical = ⌈α · n·d·b/C⌉

Computed Values (A100, 40 MB L2)

Dataset |H| (MB) L2 Coverage k*
ogbn-arxiv 86 47% 3
ogbn-products 960 4% 24

🚀 vs. Grid Search

Replaces blind sweep over k ∈ {4, 8, 16, 32, 64} with a single principled hardware-derived value. Adapts automatically to any GPU.

  • 2.4M nodes, 61.8M edges
  • 100-dim features, ~960 MB total
  • L2 coverage: 4% — nearly all cache-cold
  • Training: mini-batch (NeighborLoader, 2-hop [25,10])

GNN Models

GraphSAGE

Fixed-size neighborhood sampling. Reordering improves spatial coherence of sampled neighbor indices.

GAT (Graph Attention Network)

Computes attention coefficients αᵢⱼ → 2× neighbor feature accesses per edge (attention + weighted sum). Doubles cache pressure — amplifies reordering gains.

⚠️ Hardware Counter Restriction

Shared A100 server enforces RmProfilingAdminOnly=1 — ncu L2 hit rate profiling is blocked. We adopt two software proxy metrics instead.

📊 Proxy: Analytical Cache Coverage (ACC)

ACC = min(1, C / (n·d·b))

Upper bound on L2 hit rate under perfect spatial locality.

📊 Proxy: Temporal Reuse Ratio (TRR)

TRR = (total − unique) / total fetches

Fraction of neighbor fetches that reuse a previously accessed row. Directly trackable in software.

🔍 Key Finding: GAT Backward Dominates

GAT backward (22.4ms) > forward (16.8ms) due to per-edge attention backprop — makes GAT the stronger test case for locality-improving reordering.

Summary

What We're Doing & Why It Matters

🧱 Problem

GNN training on GPUs bottlenecked by poor L2 cache locality during sparse message-passing (SpMM). Random node ordering → near-zero cache hit rate at scale.

💡 Approach

Lossless node reordering (permutation π) to cluster connected nodes at nearby indices. Tested via RCM and METIS baselines + 2 novel methods.

🚀 Novel Contributions

Hybrid π: METIS partitioning → intra-block RCM for block + band diagonal structure.
k*: Analytically derive optimal partition count from GPU L2 capacity.

📊 Progress

Phase 1 complete: baselines measured on real A100 hardware. RCM implemented. Phases 2–5 ongoing — full ablation across 5 configs on 2 datasets × 2 models.
